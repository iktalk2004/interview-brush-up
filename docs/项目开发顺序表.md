# 项目开发顺序表

## 总体原则

- 先后端后前端：每个模块先完成后端API，再开发前端页面
- 先核心后辅助：先跑通核心刷题流程，再补充推荐、统计等增强功能
- 先简单后复杂：简答题先于代码题，本地评分先于API评分
- 每个步骤完成后进行测试验证，确认通过再进入下一步
